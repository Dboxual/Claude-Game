WHAT CHANGED IN THIS LOCAL WIP
------------------------------
Identity shift: from world-first (Play -> Worlds) to a CROSS-PLATFORM GAME HUB
where "Game Modes" are the games inside the game.

- NEW shared module src/shared/game_mode.{h,cpp}: a GameMode = ruleset + map
  template + loadout, with Playable/Preview status + honest preview notes.
  Six modes registered: MiniCS, Deathmatch, Sandbox, Zombies, Parkour,
  Social Hub.
- NEW map templates in src/shared/world_template.cpp:
    minics_arena  (walled FPS arena: raised mid, crate cover, bot opponents)
    zombie_pit    (walled survival pit seeded with goblin "horde" stand-ins)
- Menu state machine rebuilt in src/client/game.cpp + game.h:
    Play  -> Game Modes / Servers / Recent Games
    Create-> Create Map / Edit Map / Skins / Publish
    Profile, Settings, Quit
    Single goBack() drives ESC and every Back button.
    Game Modes render as cards (name, tagline, PLAYABLE/EARLY badge).
- startGameMode(): builds the mode's map, applies loadout, drops you in;
  Preview modes show an honest on-entry note.
- Presentation: hub title + accent underline, in-world mode banner,
  mode-aware control hint (gun modes hide melee/build keys; the build/spawn
  menu is now a Sandbox-only feature, off the normal player path), skin
  swatch picker placeholder, Profile/Publish placeholders.
- Guns/tracers: slimmer + longer bullet tracer with a bright head, tighter
  impact puff (less of a screen-filling blob).
- Dev flags added in src/client/main.cpp: --mode <id> and --menu <screen>
  (for screenshot/automated testing; existing --world/--equip/--fire-at/
  --screenshot/--frames/--novsync still work).
- Docs consolidated: claude_handoff.txt + claude_status.txt + CODEX_REVIEW.md
  -> one HANDOFF.md. README identity/menu sections refreshed.

Build: client + dedicated server compile on macOS arm64 with ZERO warnings
(-Wall -Wextra). All six modes and the server were smoke-tested; menus and
gameplay verified via screenshots.

WHAT STILL FEELS UNFINISHED
---------------------------
- No real networking of any kind yet (see below).
- Deathmatch / Zombies / Parkour / Social are "Early": they drop you into a
  real working world but have no scoring / wave system / run timer / online.
- Skins / Publish / Profile are placeholders (swatch preview, honest "coming
  with online" copy, last-played readout). Nothing persisted yet.
- Player health still lives client-side (playerHealth_ in game.cpp); must
  move server-side when networking lands.
- Dedicated server is config-only: it does NOT build worlds from templates,
  load server/content/, or simulate combat yet (shared modules are ready).
- World/viewmodels are still box-art (no texture/model pipeline).

BROKEN SYSTEMS
--------------
- Nothing is known-broken/crashing. All six modes boot, run 120 frames, and
  exit 0; the dedicated server runs --ticks 256 clean.
- "Broken" only in the sense of not-yet-built: multiplayer, mode rulesets
  (scoring/waves/timers), skins/publish persistence, server-side sim.

CURRENT STATE OF MULTIPLAYER
----------------------------
Not implemented; nothing is faked. Play -> Servers has:
  Server List     -> honest "NO SERVERS FOUND / discovery needs networking"
  Direct Connect  -> address box; CONNECT reports "connection system not
                     implemented yet - networking is a later milestone"
  Host Local Server -> honest "hosting needs the networking milestone"
src/shared/protocol.h only reserves a protocol version. No connection path,
no remote players, no replication, no networked health/death. See
ONLINE_SERVERS.md for the master-server/discovery checklist.

CURRENT STATE OF GUNS / TRACERS / ITEMS / MODELS
------------------------------------------------
- Guns: Glock is a working hitscan pistol (muzzle flash, recoil, hitmarker,
  damages dummies/bots). MiniCS/Deathmatch/Zombies start you with it.
- Tracers: a slim, longer amber streak that leaves the muzzle and travels to
  the impact point, with a bright head and a small impact puff. Reworked this
  pass so firing along your own view ray reads as a dash, not a fat blob.
  (Still not a real beam/particle system - box-composed.)
- Items/weapons: fists / karambit / sword (melee) + glock (hitscan); pickups,
  carry/drop props, gather+craft (RPG sandbox loop) all still work.
- Models: everything is still axis-aligned box art (world, entities,
  first-person viewmodels). No texture/model pipeline yet - looks placeholder
  by design.
