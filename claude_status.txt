TacMove — WIP status (branch wip-m3-content-platform, 2026-07-05, update 6)
============================================================================

Sixth drop: the POLISH pass. Direction changed - no more platform/PvP
feature chasing; make what exists feel like a real playable game. This
drop touches presentation, feel, and first-run experience only; no new
systems, one system deliberately simplified.

DONE THIS DROP (built on Windows x64, client + server, /W4 zero warnings)
--------------------------------------------------------------------------
POLISHED:
- The test arena is furnished (Game::startTestWorld): a weapon rack on the
  crate row (sword/glock/karambit, one per step height), a shield nearby,
  a training corner by the strafe pillars (training dummy + duelist bot),
  a crate stack and barrels near spawn to carry around. Arena-only respawn
  overrides (pickups 15 s, bots 10 s) keep the playground stocked; created
  worlds keep the nothing-respawns default. Opening the game now demos
  every system with zero spawn-menu knowledge.
- Sky: vertical gradient (deep blue zenith -> fog-colored horizon) drawn
  behind the world via the rect pipeline (GLRenderer::drawSky). The void
  above the arena walls finally reads as sky, and the fog line blends
  into it instead of hitting a flat clear color.
- Glock feel: bullet tracers (bright thinning line from muzzle to impact,
  60 ms, world-space oriented boxes) + impact puff sparks wherever the
  shot lands + slight per-shot pitch variation. Shots visibly travel and
  hit something now.
- Viewmodel feel: look sway (hands lag a beat behind the camera and
  settle), landing dip scaled by fall speed, both on the shared root so
  every weapon gets them. Footsteps got pitch variation (0.92-1.08) and
  landing plays a low thud - IAudio::playSound grew an optional pitch
  parameter (SDL_SetAudioStreamFrequencyRatio under the hood).
- Prop carrying: the carried prop now pulls in toward you when a wall is
  closer than the hold distance (raycast clamp) - it can no longer stick
  through geometry while carried. Dropping tries several spots from the
  hold point inward, settles each candidate onto the floor/stack below,
  and only places where nothing overlaps (walls, props, or the player);
  if nowhere fits you keep carrying and get told "NO ROOM TO DROP IT
  HERE". Nothing can be wedged into level geometry anymore. Quit-to-menu
  still force-drops safely if the search fails.
- First-run clarity: the debug HUD now DEFAULTS OFF (F1 brings it back;
  existing settings.cfg files keep whatever they had). In its place, a
  clean 8-second control hint fades in at the bottom on every spawn:
  "WASD MOVE  E INTERACT  LMB ATTACK  RMB BLOCK  B SPAWN MENU  ESC MENU".
  Stamina bar only shows while a melee weapon is held (the pistol never
  spends it). Main-menu subtitle is now "FIRST-PERSON SANDBOX PROTOTYPE"
  and the footer says plainly: local play, protocol reserved.

SIMPLIFIED / HIDDEN (half-baked things that made it feel worse):
- The 5-tab server-browser shell (Online/Favorites/LAN/History/Direct
  Connect) is GONE from the UI. Four permanently-empty tabs of "nothing
  here yet" made the game feel more unfinished, not more ambitious. The
  Multiplayer screen is now ONE honest screen: server address box,
  Connect (still reports the connection system does not exist),
  localhost shortcut, and a single line - "online play arrives with the
  networking milestone". The removed shell is trivial to resurrect from
  git history when real discovery exists; ONLINE_SERVERS.md (the design
  doc for that milestone) is untouched. mpTab_ and the SelectMpTab
  button id were deleted outright, not commented out.

WHERE THE NUMBERS LIVE (for taste tuning)
-----------------------------------------
- Arena layout: Game::startTestWorld (game.cpp) - place() calls.
- Sky colors: GLRenderer::drawSky (gl_renderer.cpp) - t[] / b[].
- Sway/dip/bob strengths: the rootPos block in appendViewmodelDraws.
- Tracer look: the Tracer block in tryAttack + the draw in
  buildRenderFrame.
- Footstep/land pitches: the tick loop in Game::update.

UNFINISHED (known, deliberate)
------------------------------
- No player health; duel bot hits remain feedback-only training.
- Melee kept in full (it is stable and tested); its controls are the
  richest part of the game - the spawn hint plus F1 cover discovery.
- Server binary still config-only; combat/sim wiring is the networking
  milestone. Maps still code-built; music slider still drives nothing.
- Load World list still caps at 7; fine until someone has 8 worlds.

BROKEN / WATCH OUT
------------------
- IAudio::playSound signature changed (added pitch with a default) - any
  future IAudio implementation must match the two-argument form.
- config/settings.cfg is gitignored but persists locally: your existing
  file still has show_debug_hud from before, so YOU may still see the
  debug HUD while fresh installs will not. Delete the file or press F1.
- The test arena now adds entities every time it starts; if kMaxEntities
  (256) ever feels tight in the arena, the furniture is the first 11.
