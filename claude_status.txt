TacMove — WIP status (branch wip-m3-content-platform, 2026-07-05, update 5)
============================================================================

Fifth drop on this branch: the melee rework. Update 4 built the skeleton
(stamina/phases/parry); this drop makes it FEEL like a duel, following the
Chivalry 2 mechanic STRUCTURE (read -> react -> punish -> stamina pressure
-> finish) with entirely original values, names, code, sounds and visuals.
Written by Claude on Windows 11 (MSVC, RTX 4060 Ti).

DONE THIS DROP (built on Windows x64, client + server, /W4 zero warnings)
--------------------------------------------------------------------------
- THE BLADE TRAVELS THROUGH SPACE (the big feel fix): during the release,
  the strike ray is no longer the crosshair - it SWEEPS. Slashes arc +/-32
  degrees across the view (left slash crosses left->right, right slash the
  mirror), overheads sweep +28 -> -22 degrees top to bottom, stabs/jabs
  thrust straight. Contact lands where the blade actually is, sparks and
  all. Because the sweep is camera-relative this IS drag/accel: turning
  into your swing connects earlier, dragging away delays it. ~20 lines in
  performMeleeStrike (game.cpp), zero new systems.
- Commitment / no free spam: whiffed swings recover x1.2 (kicks x1.5,
  melee.cpp tick); swinging into walls, crates or barrels now CLANGS -
  sound, sparks, bounce recovery (melee::bounceOff) instead of a silent
  no-op. Combo chains still cost escalating stamina.
- JAB (V key, new Attack::Jab): 0.13 s windup pommel bash, 3 damage,
  7 stamina, 0.9 s cooldown, 1.5 m. Interrupts windups (new outcome
  Outcome::Interrupted, 0.55 s flinch) to stop pressure and reset tempo;
  just thuds into guards (cheap 4-stamina block for the defender). Any
  clean hit on a winding-up defender now reports Interrupted, so normal
  swings also read as "you got interrupted" both ways.
- RIPOSTE is now visibly a reward: a big gold "RIPOSTE!" prompt sits above
  the crosshair for the whole 0.9 s window, and ripostes are PROTECTED -
  strikes into a riposte windup get deflected (new Outcome::Deflected:
  attacker bounces + pays, defender loses nothing, toast explains it).
  Kicks/jabs cannot interrupt a riposte either (they fall through).
- Counter language is taught, not guessed: aiming at the bot during its
  windup names the attack ("WINDING UP STAB" in the readout). Answer with
  the same attack type -> PERFECT COUNTER (unchanged rules from update 4:
  attacker staggered 1.3 s, your windup accelerates into a riposte).
- Stamina pressure: holding the guard up now drains 2/s (3/s with shield)
  on top of pausing regen - turtling loses the stamina war. Per-attack
  block costs moved into the spec table (slash 12 / overhead 16 / stab 10 /
  jab 4, heavies x1.6, shield x0.6, low-stamina x1.5). Timings slowed to
  duel weight: sword slash 0.35 windup / 0.18 release / 0.42 recover,
  overhead 0.42/0.16/0.48, stab 0.30/0.12/0.38 (x weapon weight).
- Impact reactions: damageable entities FLINCH - the visual-part stack
  leans away from the player on a hit (taller parts lean further, so the
  whole thing tips) and a staggered duelist reels back for the entire
  stagger. Combined with hit-stop + the new deeper melee_hit (150->85 Hz
  pitch-dropping thump regenerated in tools/generate_sounds.cpp), clean
  hits finally land ON something.
- Bot timing polish: its strike now lands 35% into the hit window (the
  blade has visibly turned red by then), giving honest parry timing.
  Recolored bronze (builtin + duel_bot.cfg) because the old slate armor
  camouflaged against the arena walls - reading the bot IS the gameplay.
- RESPAWN DEFAULTS FIXED (the second half of the task): respawn_seconds
  is now 0 for EVERYTHING - weapon pickups, shield, training dummy, duel
  bot (builtins + all cfg files). Taken/destroyed means gone unless a def
  or a per-entity world-save line opts in. Thrown/dropped weapons already
  spawned with respawn 0 and stay in the world until re-taken. Docs
  updated (content.h comment, server/README.md, README.md).

HOW TO TEST (2 minutes; also in README "Melee combat")
------------------------------------------------------
  1. run_windows.bat -> Singleplayer -> Start Test World (or --equip sword)
  2. B -> BOTS -> DUELIST BOT, B to close. Aim at it: readout shows HP,
     stamina, and what it is winding up.
  3. Parry: tap RMB just before its red-blade moment -> RIPOSTE! prompt ->
     click = fast protected riposte.
  4. Counter: it winds up STAB (readout says so) -> wheel-down yourself as
     the strike lands -> PERFECT COUNTER.
  5. Jab: V the instant it starts winding -> "INTERRUPTED ITS ATTACK".
  6. Kick: F while it holds its guard -> "GUARD KICKED OPEN" -> free hit.
  7. Drag: start a slash aimed left of the bot, sweep your mouse across it
     during the release - contact follows the blade, not the click.
  8. Feel the commitment: whiff on purpose, swing into a crate, spam kicks.
  9. Throw (Q), pick it back up with E - it stays in the world forever now.

UNFINISHED (known, deliberate)
------------------------------
- No player health yet: bot hits are feedback (flash/shake/sound), not
  death. Wire player HP + death/respawn when the mode needs it.
- Bots do not move, feint, jab, kick, parry, or counter. Training partner
  only; movement/AI is its own milestone.
- Drag/accel affects contact TIMING only (the swept ray). No turn-cap or
  windup rotation limits yet; add if real PvP shows ballerina spins.
- Combat tuning constants live in melee.h, not data files, until balance
  settles. Sound-event mapping still client code.
- Server binary still does not simulate combat (module is shared + pure
  and ready; lands with server-side simulation).

BROKEN / WATCH OUT
------------------
- Two orphaned tacmove.exe instances (user playtest leftovers) held the
  exe lock this session and had to be killed before linking. If a build
  fails with LNK1104 on tacmove.exe, check Task Manager first.
- The sweep means point-blank walls/props on your swing side eat the hit
  early (by design - clang). If a future arena is very cramped, melee near
  walls will feel restrictive; that is Chivalry-authentic but tunable via
  the arc width.
- resolveStrike outcome list grew (Interrupted, Deflected): any new caller
  must handle them or the switch will not compile (deliberate - no default
  case, so the compiler enforces completeness).
